Data Modeling

  • First step in database design and object-oriented programming

  • Goal = create conceptual model of
    • What data are
    • How data relate to each other
  • Process -
    • Conceptual – high-level domain models
    • Logical – describe logical entities & relationships
    • Physical – detailed; used to design database schemas

Entity Characteristics

  • Entity is a single data point (datum)
  • Individual characteristics:
    • Physical
      • Height
      • Hair color
      • Eye color
    • Arbitrary
      • ID numbers
        • SSN
        • DL Number
    • Internal
      • Motivation
      • Personality

Relationship Characteristics

  • Entity-to-Entity Relationship
  • Relationship characteristics:
    • Type of Relationship
      • Sibling
      • Nemesis
      • Friend
    • Duration

Relationship Characteristics – Cardinality

Key characteristic of relationship

  • How many?
    • 1:1 (One to one)
      • Each entity in first group matches exactly one entity in second group.
      • Loki:Thor
    • 1:∞ (One to many)
      • Each entity in first group may have many entities in second group.
      • Loki:Avengers
    • ∞:∞ (Many to many)
      • Each entity in first group may have many entities in second group.
      • Each entity in second group may have many entities in first group.
      • Chitauri:Avengers

Grouping Entities

  • Entities will share several characteristics with other entities
    • Appearance
    • Characterization
  • Similar entities grouped in tables
    • Each entity = record/row
    • Each attribute = field/column
    • Unique identifier = Primary Key
      • Composite key

Relationships

  • Connection between tables
  • Primary Key = Foreign Key
  • FK column
    • Hint relationship type (column name)
    • Cardinality (restrictions)

Entity-Relationship Diagram (ERD)

  • Entities
    • Attributes/characteristics
  • Relationships
    • Cardinality
